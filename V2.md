> *Authored from the combined lens of bathymetrist, senior software engineer, and physical oceanographer.*

## Executive Summary

Peakcast V1 delivers personalized surf scoring on top of Open-Meteo + NDBC buoy data with a physics-based bias corrector. This is a solid foundation but uses the same underlying model data as every other app (ECMWF IFS via Open-Meteo, NDBC stdmet). V2's goal is to build a **genuinely superior forecasting system** that no commercial competitor currently offers at any price:

1. Full directional spectral wave analysis (not Hs/Tp summaries)
2. WAVEWATCH III local ingestion → per-spot swell train tracking
3. CDIP directional buoy fusion (UCSD's research-grade network)
4. High-resolution bathymetry + per-spot SWAN refraction LUTs
5. Probabilistic forecasts with true uncertainty quantification
6. Session-adaptive ML that gets smarter with each surf logged
7. Computer vision wave height validation from public surf cams

**Target accuracy improvement**: Reduce face-height RMSE from ~0.35m (V1 physics fallback) to <0.12m (V2 full pipeline). Outperform Surfline's CAL1 model on every spot where ground truth exists.

### 1.1 CDIP — Coastal Data Information Program (Priority: CRITICAL)
**Why**: CDIP is the highest-quality publicly available wave observation network. Surfline pays for access; it is free for anyone via API. Provides **full directional energy spectra** at 0.02 Hz resolution (vs NDBC stdmet's scalar Hs/Tp only).

**Source**: https://cdip.ucsd.edu/m/products/?type=nowcast

- `waveTailFactor`, `waveBandwidth` — spectral metadata
- `sstSeaSurfaceTemperature` — 15-min SST observations
- `acmCurrentSpeed`, `acmCurrentDirection` — near-surface current vector (selected buoys)

```python
import xarray as xr
import numpy as np

CDIP_THREDDS = "https://thredds.cdip.ucsd.edu/thredds/dodsC/cdip/realtime"

async def fetch_cdip_spectrum(station_id: str) -> dict:
    """
    Returns directional wave spectrum as frequency × energy dict.
    Spectrum resolution: 0.025–0.580 Hz in 64 bins.
    """
    url = f"{CDIP_THREDDS}/{station_id}p1/{station_id}p1_rt.nc"
    ds = xr.open_dataset(url, engine="pydap")

    freq = ds["waveFrequency"].values
    energy = ds["waveEnergyDensity"].values[-1]  # latest observation
    direction = ds["waveMeanDirection"].values[-1]

    return {
        "frequencies": freq.tolist(),
        "energy": energy.tolist(),
        "direction": direction.tolist(),
        "timestamp": str(ds["waveTime"].values[-1]),
        "station_id": station_id,
    }
```

### 1.2 WAVEWATCH III (WW3) — NOAA Global Wave Model (Priority: HIGH)
**Why**: WW3 is the gold-standard global wave model. NOAA distributes 7-day hourly global forecasts at 0.5° (30nm) and regional grids at 0.05° (3nm) for US coasts. Unlike Open-Meteo which gives Hs/Tp summary only, WW3 distributes **full 2D directional energy spectra** at each grid point — the same data Surfline's WaveWatch ingestion uses.

**Source**: NOAA NOMADS server (free, no key required)
**Endpoint**: `https://nomads.ncep.noaa.gov/pub/data/nccf/com/wave/prod/`
**Format**: GRIB2 (cfgrib/eccodes-compatible) or NetCDF

**Available grids**:
- `multi_1.glo_30m` — Global 0.5° (use for distant storm tracking)
- `multi_1.at_10m` — North Atlantic 0.1° (East Coast spots)
- `multi_1.wc_10m` — West Coast 0.1° (California spots)
- `multi_1.ep_10m` — Eastern Pacific 0.1° (Hawaii)

**Key WW3 spectral fields**:
- `EFTH` — directional wave energy spectrum (m²·Hz⁻¹·rad⁻¹), 24 freq × 24 dir bins
- `HS` — significant wave height (m)
- `TP` — peak period (s)
- `PDIR` — peak direction (deg)
- `SWELL[1-3]` — up to 3 partitioned swell components (Hs, Tp, Dir each)

**Swell train tracking algorithm**:
```python
# ml/swell_tracker.py — new V2 module
from scipy.signal import find_peaks
import numpy as np

def partition_spectrum(freq: np.ndarray, energy_2d: np.ndarray) -> list[dict]:
    """
    Partition WW3/CDIP spectrum into discrete swell trains.
    Algorithm: JONSWAP fitting + watershed segmentation on 2D E(f,θ).
    Returns list of {"Hs": float, "Tp": float, "direction": float, "spread": float}
    """
    # 1. Find spectral peaks (local maxima in freq domain)
    total_energy = energy_2d.sum(axis=1)  # integrate over directions
    peaks, properties = find_peaks(total_energy, height=0.01, distance=3)

    swell_trains = []
    for peak_idx in peaks:
        peak_freq = freq[peak_idx]
        peak_period = 1.0 / peak_freq

        # Integrate energy around peak (±3 bins)
        band = slice(max(0, peak_idx-3), min(len(freq), peak_idx+4))
        band_energy = energy_2d[band, :].sum()
        Hs = 4 * np.sqrt(band_energy)  # significant height from spectral moment m0

        # Peak direction (integrated over band)
        dir_weights = energy_2d[band, :].sum(axis=0)
        direction = np.average(np.arange(0, 360, 15), weights=dir_weights)
        spread = np.sqrt(np.average((np.arange(0, 360, 15) - direction)**2, weights=dir_weights))

        swell_trains.append({
            "Hs": round(Hs, 2),
            "Tp": round(peak_period, 1),
            "direction": round(direction, 0),
            "spread": round(spread, 1),
            "peak_freq": peak_freq,
        })

    return sorted(swell_trains, key=lambda x: x["Hs"], reverse=True)
```

**Storm origin tracking**: Use WW3 global 0.5° grid to trace swell energy back to generating storm. Compute great circle path, estimate travel time to spot:
```
travel_time_hours = great_circle_distance_km / (group_velocity_km_h)
group_velocity = (g × T) / (4π) = (9.81 × Tp) / (4π × 3.6) km/h ≈ 0.77 × Tp km/h
```
A 16-second swell travels ~12 km/h → Tasman Sea to Hawaii in ~6 days.

## Part 2: ML Model Improvements

### 2.1 V2 Bias Correction — Physics-Informed Neural Network (PINN)
**Problem with V1**: LightGBM learns a static correction from bootstrap labels derived from the physics formula. It's learning a noisy version of the formula it was bootstrapped from. This is circular and caps accuracy.

**V2 approach**: Physics-Informed Neural Network that embeds shallow water wave equations as hard constraints, learning only the residual:

```python
# ml/train_pinn_bias.py
import torch
import torch.nn as nn

class WaveTransferPINN(nn.Module):
    """
    Physics-Informed Neural Network for wave height transfer.

    Physics constraints embedded:
    1. Shoaling coefficient: K_s = sqrt(C_g_deep / C_g_shallow)
    2. Refraction coefficient: K_r = sqrt(b_deep / b_local) (crest spacing ratio)
    3. Breaking criterion: H_b < 0.78 × h (Miche breaking limit)
    4. Energy conservation: E × C_g = constant along wave ray (no dissipation)
    """

    def __init__(self, n_features: int = 48):
        super().__init__()
        # Shared encoder (spectral + temporal features)
        self.encoder = nn.Sequential(
            nn.Linear(n_features, 256),
            nn.SiLU(),
            nn.LayerNorm(256),
            nn.Dropout(0.1),
            nn.Linear(256, 128),
            nn.SiLU(),
            nn.LayerNorm(128),
        )
        # Physics-correction head (outputs residual corrections)
        self.physics_residual = nn.Sequential(
            nn.Linear(128, 64),
            nn.SiLU(),
            nn.Linear(64, 1),
            nn.Tanh(),  # constrain residual to [-1, 1]
        )
        # Confidence head
        self.confidence_head = nn.Sequential(
            nn.Linear(128, 32),
            nn.SiLU(),
            nn.Linear(32, 1),
            nn.Sigmoid(),
        )

    def physics_baseline(
        self,
        buoy_hs: torch.Tensor,    # (B,)
        period: torch.Tensor,      # (B,) in seconds
        angle_diff: torch.Tensor,  # (B,) in degrees
        depth_ratio: torch.Tensor, # (B,) h_local / lambda_deep
    ) -> torch.Tensor:
        """
        Compute physics baseline using linear wave theory.
        """
        # Shoaling: Ks = sqrt(C_g_deep / C_g_shallow)
        # Using linear dispersion: C_g_deep = g*T/(4π), simplified for deep water
        g = 9.81
        omega = 2 * torch.pi / period
        k_deep = omega**2 / g  # deep water wavenumber

        # Shallow water approximation for nearshore
        Ks = (1 + 2 * k_deep * depth_ratio).pow(-0.25)  # first-order shoaling

        # Refraction: cos(angle_diff) approximation
        Kr = torch.cos(torch.deg2rad(angle_diff.clamp(0, 80)))

        # Combined physics estimate
        return buoy_hs * Ks * Kr * 0.85  # 0.85 = face-height Hawaiian factor

    def forward(self, x: torch.Tensor, physics_inputs: dict) -> dict:
        # Physics baseline
        baseline = self.physics_baseline(**physics_inputs)

        # Neural correction
        encoded = self.encoder(x)
        residual = self.physics_residual(encoded).squeeze(-1)
        confidence = self.confidence_head(encoded).squeeze(-1)

        # Final prediction: physics + learned residual
        # residual scale: ±0.5m correction max
        face_height = baseline + residual * 0.5
        face_height = torch.clamp(face_height, 0.1, 10.0)

        return {"face_height": face_height, "confidence": confidence, "baseline": baseline}

# Loss function: combination of MSE + physics violation penalties
def pinn_loss(pred: dict, target: torch.Tensor, physics_inputs: dict) -> torch.Tensor:
    # Data loss
    mse = nn.functional.mse_loss(pred["face_height"], target)

    # Physics regularization: residual should be small (trust physics first)
    residual_penalty = 0.1 * (pred["face_height"] - pred["baseline"]).pow(2).mean()

    # Breaking criterion: H <= 0.78h (prevents physically impossible predictions)
    depth_m = physics_inputs["depth_ratio"] * 50  # approximate depth (m)
    breaking_violation = torch.relu(pred["face_height"] - 0.78 * depth_m).pow(2).mean()

    return mse + residual_penalty + 0.5 * breaking_violation
```

### 2.2 V2 Peak Score — Session-Adaptive ML with Online Learning

**V1 problem**: Static weights (0.30/0.20/0.25/0.15/0.10) are the same for every user forever. No learning.

**V2 approach**: Bayesian personalization — start with global population priors, update per-user from session ratings.

```python
# services/stoke_score_v2.py
import numpy as np
from scipy.stats import norm

class BayesianStokePersonalizer:
    """
    Per-user Bayesian weight updater.
    Prior: population weights from global model.
    Posterior: updated from user's session ratings.
    """

    # Global population priors (mean, variance) for each weight
    PRIORS = {
        "height":    (0.30, 0.05),  # μ, σ²
        "period":    (0.20, 0.04),
        "direction": (0.25, 0.04),
        "wind":      (0.15, 0.03),
        "crowd":     (0.10, 0.03),
    }

    def __init__(self, user_id: str, user_sessions: list[dict]):
        self.weights = {k: list(v) for k, v in self.PRIORS.items()}
        if user_sessions:
            self._update_from_sessions(user_sessions)

    def _update_from_sessions(self, sessions: list[dict]):
        """
        Bayesian update: use session ratings as observations.
        Each session: {"quality_rating": 1-10, "component_scores": {...}}
        """
        for session in sessions:
            if not session.get("quality_rating") or not session.get("component_scores"):
                continue

            observed_quality = session["quality_rating"] / 10.0
            components = session["component_scores"]  # each 0-1

            # Gradient: which component best explains the observed rating?
            # Update weights to minimize squared error
            predicted = sum(self.weights[k][0] * components.get(k, 0.5) for k in self.weights)
            error = observed_quality - predicted

            for k in self.weights:
                # Simplified conjugate Gaussian update
                c_k = components.get(k, 0.5)
                lr = self.weights[k][1] / (self.weights[k][1] + 0.01)  # Kalman-like gain
                self.weights[k][0] += lr * error * c_k
                self.weights[k][1] *= (1 - lr)  # variance shrinks with more data

            # Renormalize weights to sum to 1
            total = sum(w[0] for w in self.weights.values())
            for k in self.weights:
                self.weights[k][0] /= total

    def get_weights(self) -> dict[str, float]:
        return {k: v[0] for k, v in self.weights.items()}

    def get_uncertainty(self) -> dict[str, float]:
        """Higher variance = less confident in this user's preference for component."""
        return {k: v[1] for k, v in self.weights.items()}
```

**Additional stoke features**:
- `swell_spread_deg`: Tight swell (< 20°) → clean peeling waves; messy (> 45°) → choppy
- `secondary_swell_ratio`: If secondary swell > 40% of primary → confused sea state
- `wind_consistency`: Standard deviation of wind direction last 3h (gusty = bad)
- `tidal_window_score`: Per-spot reef/sand optimal tide stage (LUT from SWAN sensitivity analysis)

### 2.3 V2 Ensemble — Regional Model Weighting with Temporal Decay

**V1 problem**: Fixed weights (ECMWF 45%, GFS 30%, ICON 25%) globally, no temporal skill decay.

**V2**: Dynamic weights based on regional historical skill + forecast horizon decay.

```python
# services/ensemble_v2.py

# Regional skill lookup (computed from ERA5 hindcast validation, per season)
MODEL_SKILL_BY_REGION = {
    # format: (region, season) → {model: rmse_vs_ERA5}
    ("North Pacific", "winter"): {"ecmwf_ifs": 0.18, "gfs": 0.25, "icon": 0.29},
    ("North Pacific", "summer"): {"ecmwf_ifs": 0.22, "gfs": 0.28, "icon": 0.31},
    ("North Atlantic", "winter"): {"ecmwf_ifs": 0.16, "gfs": 0.22, "icon": 0.20},
    ("North Atlantic", "summer"): {"ecmwf_ifs": 0.20, "gfs": 0.25, "icon": 0.22},
    ("Tropics", "*"):             {"ecmwf_ifs": 0.25, "gfs": 0.23, "icon": 0.35},
    ("Southern Ocean", "*"):     {"ecmwf_ifs": 0.19, "gfs": 0.28, "icon": 0.24},
}

# Temporal skill decay: models degrade after day 5
SKILL_DECAY = {
    "ecmwf_ifs": lambda day: max(0.1, 1.0 - 0.08 * max(0, day - 4)),
    "gfs":       lambda day: max(0.1, 1.0 - 0.10 * max(0, day - 4)),
    "icon":      lambda day: max(0.1, 1.0 - 0.12 * max(0, day - 3)),
}

def compute_dynamic_weights(region: str, season: str, forecast_day: int) -> dict[str, float]:
    skill = MODEL_SKILL_BY_REGION.get((region, season)) or MODEL_SKILL_BY_REGION.get((region, "*"), {})
    if not skill:
        skill = {"ecmwf_ifs": 0.20, "gfs": 0.25, "icon": 0.28}  # default

    # Inverse-RMSE weighting × temporal decay
    raw_weights = {
        m: (1 / rmse) * SKILL_DECAY[m](forecast_day)
        for m, rmse in skill.items()
    }
    total = sum(raw_weights.values())
    return {m: w / total for m, w in raw_weights.items()}
```

### 2.4 V2 SWAN — Automated Multi-Spot Deployment with LUT Pre-Computation

**V1 problem**: SWAN runs only at runtime (6h cadence, slow). Not feasible for 200+ spots.

**V2 approach**: Pre-compute refraction Look-Up Tables (LUTs) offline. At runtime, interpolate into LUT instead of running SWAN.

```
Offline (monthly or on bathy change):
  For each spot + SWAN grid:
    For each (Hs, Tp, direction, tide) combination in parameter space:
      Run SWAN → record (face_height, spread, setup)
    Save LUT as HDF5: shape [n_hs, n_tp, n_dir, n_tide] → [face_height, spread]

Online (per forecast hour):
  f(Hs=1.2m, Tp=14s, dir=285°, tide=0.8m) → interpolate LUT → face_height=1.85m, spread=28°
  Runtime: < 1ms per spot (vs 5-30min per SWAN run)
```

**LUT parameter space**:
```python

# ml/build_swan_lut.py
import asyncio
import h5py
import numpy as np
from itertools import product

async def build_lut_for_spot(spot_slug: str, concurrency: int = 4):
    """
    Build full SWAN LUT for a spot. Runs in background on NUC.
    Saves to: data/swan_luts/{spot_slug}.h5
    """
    Hs_vals = np.arange(0.25, 6.0, 0.25)
    Tp_vals = np.arange(6, 22, 1)
    dir_vals = np.arange(0, 360, 15)
    tide_vals = np.arange(-1.5, 2.5, 0.25)

    shape = (len(Hs_vals), len(Tp_vals), len(dir_vals), len(tide_vals))
    lut = np.zeros(shape, dtype=np.float32)
    spread_lut = np.zeros(shape, dtype=np.float32)

    sem = asyncio.Semaphore(concurrency)

    async def run_cell(i, j, k, l):
        async with sem:
            result = await run_swan_boundary(
                spot_slug=spot_slug,
                Hs=Hs_vals[i], Tp=Tp_vals[j],
                direction=dir_vals[k], tide=tide_vals[l]
            )
            lut[i, j, k, l] = result.get("Hs_local", 0)
            spread_lut[i, j, k, l] = result.get("spread_deg", 45)

    tasks = [run_cell(i, j, k, l)
             for i, j, k, l in product(*[range(s) for s in shape])]
    await asyncio.gather(*tasks)

    with h5py.File(f"data/swan_luts/{spot_slug}.h5", "w") as f:
        f.create_dataset("face_height", data=lut, compression="gzip")
        f.create_dataset("spread_deg", data=spread_lut, compression="gzip")
        f.attrs["Hs_axis"] = Hs_vals
        f.attrs["Tp_axis"] = Tp_vals
        f.attrs["dir_axis"] = dir_vals
        f.attrs["tide_axis"] = tide_vals
        f.attrs["spot_slug"] = spot_slug
```

### 2.5 V2 Probabilistic Forecasting — Deep Ensemble with Prediction Intervals

**Why**: Every surf forecast app gives a single number. Peakcast V2 will show **"85% chance 2–4ft"** — a quantile forecast that communicates real uncertainty. Bayesian surfers (every advanced surfer) will pay for this.

**Architecture**: Deep Ensemble of 5 PINNs trained with different random seeds. Prediction interval from empirical distribution.

```python
# services/probabilistic_forecast.py

class ProbabilisticWaveEnsemble:
    def __init__(self, spot_slug: str, n_models: int = 5):
        self.models = [
            WaveTransferPINN.load(f"models/ml/pinn_{spot_slug}_{i}.pt")
            for i in range(n_models)
        ]

    def predict_with_intervals(self, features: dict, alpha: float = 0.1) -> dict:
        """
        Returns (1-alpha) prediction interval.
        Default: 90% interval (alpha=0.1 → 5th–95th percentile).
        """
        predictions = []
        for model in self.models:
            with torch.no_grad():
                pred = model(features)
                predictions.append(pred["face_height"].item())

        predictions = sorted(predictions)
        n = len(predictions)
        lower_idx = int(np.floor(alpha/2 * n))
        upper_idx = int(np.ceil((1-alpha/2) * n)) - 1

        return {
            "median": float(np.median(predictions)),
            "lower_90": predictions[lower_idx],
            "upper_90": predictions[upper_idx],
            "mean": float(np.mean(predictions)),
            "std": float(np.std(predictions)),
            "confidence": 1.0 - (np.std(predictions) / max(np.mean(predictions), 0.1)),
        }
```

**UI**: Timeline shows wave height as a **band chart** (shaded 90% interval + median line) rather than a single line. Color intensity = confidence.

### 2.6 V2 Multi-Buoy Swell Fusion

**Why**: No surf app uses multiple buoys simultaneously. Yet for a spot like Rincon, both the Harvest buoy (Point Conception, 120nm offshore) and the Santa Barbara buoy (30nm) contain different spectral information. Fusion improves accuracy.

```python
# services/swell_fusion.py

def fuse_buoy_observations(
    observations: list[dict],    # list of {station_id, distance_km, bearing_deg, spectrum, ...}
    spot: Spot,
) -> dict:
    """
    Inverse-distance-weighted fusion of multiple buoy observations.
    Directional correction: only energy within ±60° of spot's optimal window contributes.
    """
    fused_spectrum = np.zeros(64)  # 64 CDIP frequency bins
    total_weight = 0.0

    for obs in observations:
        distance_km = obs["distance_km"]
        bearing = obs["bearing_deg"]
        spectrum = np.array(obs["spectrum"])

        # Directional filter: does this buoy's bearing align with spot's swell window?
        angle_diff = min(abs(bearing - spot.optimal_swell_direction), 360 - abs(bearing - spot.optimal_swell_direction))
        if angle_diff > 90:
            continue  # buoy is behind the spot — ignore

        # Weight: inverse distance × directional cosine
        dir_weight = np.cos(np.radians(angle_diff))
        dist_weight = 1.0 / (distance_km ** 0.5)  # closer buoys weighted more
        weight = dir_weight * dist_weight

        # Swell attenuation by distance (energy decays as 1/r in deep water)
        attenuation = np.exp(-0.0001 * distance_km)  # 10% decay per 1000km

        fused_spectrum += spectrum * weight * attenuation
        total_weight += weight

    if total_weight == 0:
        return None

    fused_spectrum /= total_weight
    return partition_spectrum(CDIP_FREQUENCIES, fused_spectrum)  # returns swell trains
```
